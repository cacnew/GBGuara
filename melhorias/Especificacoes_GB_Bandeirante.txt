FASE 13 — Configurações Gerais da Academia
Objetivo

Criar um módulo exclusivo para usuários Administradores contendo todas as configurações globais do sistema que impactam regras de negócio, notificações e funcionamento da academia.

Este módulo deverá ser expansível, pois futuramente outras configurações serão adicionadas.

13.1 Configuração da Métrica para Graduação
Objetivo

Permitir que cada academia defina quantas aulas um aluno precisa frequentar para tornar-se apto à próxima graduação.

Importante:

O sistema NÃO gradua automaticamente.

Ele apenas identifica quando o aluno está apto segundo a quantidade de aulas cadastradas.

A graduação continua sendo decisão exclusiva do professor.

Regras

Criar uma tela chamada

Configurações → Graduação

Exibir uma tabela contendo todas as faixas existentes.

Exemplo:

Faixa Atual | Próxima Faixa | Nº de aulas para estar apto

Branca | Cinza | 40

Cinza | Amarela | 50

Amarela | Laranja | 60

...

Preta 1º Grau | 2º Grau | 365

O administrador poderá alterar qualquer valor.

Os valores devem ser salvos imediatamente.

Não permitir valores negativos.

Permitir apenas números inteiros.

Caso a academia utilize apenas algumas faixas, mostrar somente as cadastradas.

Banco

Criar tabela:

Critério de cálculo

Sempre que uma presença for confirmada:

Incrementar contador do aluno.

Comparar:

Aulas Realizadas >= QuantidadeConfigurada

Caso verdadeiro:

AlunoAptoGraduacao = true

Requisitos Gerais para Todas as Novas Fases

Todas as funcionalidades implementadas deverão seguir o padrão arquitetural existente no sistema e respeitar os seguintes requisitos:

Interface responsiva para desktop, tablet e dispositivos móveis.
Controle de permissões por perfil (Administrador, Professor e Aluno).
Registro de auditoria para todas as inclusões, alterações e exclusões.
Histórico de alterações quando aplicável.
Validação de campos obrigatórios e mensagens de erro amigáveis.
Padronização visual conforme o design system do Nexus Dojo.
Notificações configuráveis e desacopladas da regra de negócio, permitindo futura integração com WhatsApp, e-mail, push e outros canais.
Cobertura por testes unitários, testes de integração e testes end-to-end (E2E).
Performance adequada para academias com grande volume de alunos, evitando consultas desnecessárias e utilizando paginação, cache e índices quando apropriado.
Todas as novas telas devem seguir o mesmo padrão de navegação, componentes, acessibilidade e experiência do usuário já implementados nas fases anteriores.

Essas fases complementam o MVP atual e elevam o sistema de um gerenciador operacional para uma plataforma completa de gestão e relacionamento entre academia, professores e alunos.
